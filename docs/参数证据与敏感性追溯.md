# 参数—证据—敏感性追溯

## 追溯目标

追溯表将三个问题放在同一行回答：

1. 当前参数是什么；
2. 参数依据来自哪里、可信状态是什么；
3. 在当前场景附近，模型结果对它有多敏感。

参数来源状态沿用：

- `official`：公开官方信息直接支持；
- `public-estimate`：可由公开资料估计，但不是官方参数；
- `calibrated`：为匹配代理模型行为或演示观测而标定；
- `assumed`：缺少公开数据时采用的研究假设。

来源状态描述的是证据性质，不代表数值精度等级。

## 当前敏感性方法

v0.3.1 对十项关键参数进行确定性单因素双侧局部扰动：

- 等效回收质量；
- 初始垂向速度；
- 网系总刚度、总阻尼和代理强度上限；
- 无线基础时延和丢包率；
- 箭上位置噪声；
- 网中心位置增益；
- 遥测失效中止阈值。

每次实验保持相同 seed，只改变一个参数，并分别计算下调与上调结果。非零参数按相对比例扰动；零值参数采用按参数定义配置的绝对扰动下限。连续敏感性系数定义为：

```text
主导输出指标的相对变化 ÷ 输入参数的相对扰动
```

输出候选包括最小错位、捕获相对速度、峰值接触力、最大估计误差和稳定终态。分级规则为：

- 低：系数 < 0.25；
- 中：0.25 ≤ 系数 < 1；
- 高：连续系数 ≥ 1。

若任一侧扰动导致终态类别改变，则该参数标记为“模式边界”，连续敏感性系数置空。模式跳变优先于数值大小解释。

## 正确解释

该结果只描述：

- 当前标称点附近；
- 当前 seed；
- 当前代理模型和输出指标；
- 单因素、有限幅度扰动。

它不能替代全局敏感性、不确定度传播、参数相关性分析或真实型号试验。低敏感性也不等于参数真实或不重要；它可能只是当前场景没有激发对应机理。

## 可重复执行

浏览器“证据”页可以计算并显示追溯表。命令行可运行：

```powershell
npm run lab:sensitivity
npm run lab:sensitivity -- --preset low-damping --json
```

JSON 输出适合保存到实验记录或后续数据处理流程。
