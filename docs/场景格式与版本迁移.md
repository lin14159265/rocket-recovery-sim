# 场景 JSON 格式与版本迁移

## 目的

场景文件用于保存可重复实验的完整输入。它只描述本项目的公开机理代理模型，不是型号参数文件，也不应被解释为真实任务配置。

## 当前格式

v0.3.0 使用两层版本：

- `documentVersion`：外层交换文档版本，当前为 `2`。
- `config.schemaVersion`：场景配置结构版本，当前为 `2`。

浏览器导出的文件包含：

```json
{
  "format": "recovery-proxy-scenario",
  "documentVersion": 2,
  "modelVersion": "0.3.0",
  "configFingerprint": "...",
  "notice": "公开机理代理模型场景，不是官方型号参数",
  "config": {}
}
```

`configFingerprint` 用于识别配置内容是否发生变化，不是数字签名，也不提供来源真实性证明。

## 导入兼容范围

导入器接受：

1. v1 原始 `ScenarioConfig` 对象；
2. v0.2 导出的 `recovery-proxy-scenario` 文档包；
3. 当前 v2 文档包。

从 v1 迁移到 v2 时，会补入四类默认关闭的显式故障计划：无线静默、绞盘卡滞、导航偏置阶跃、推力降额。缺失的参数来源说明由当前标称场景补齐，并在界面中显示迁移警告。

## 拒绝规则

导入器会拒绝：

- 不是 JSON 对象的顶层内容；
- 未知的文档 `format`；
- 高于当前支持版本的 `schemaVersion`；
- 非有限数值、非法概率、非正步长或持续时间；
- 不受支持的算法模式或绞盘节点；
- 维度错误的三维向量；
- 闭合半间距不小于张开半间距等明显矛盾参数。

这是结构和基本物理边界检查，不等同于场景在工程上合理。

## 迁移原则

- 旧字段保留原值；新字段使用保守默认值。
- 不静默接受未来版本。
- 每次迁移返回来源版本、目标版本和中文警告。
- 导入后不会立即覆盖当前结果；用户必须点击“应用参数并重算”。
- 运行结果继续记录模型版本、配置指纹、seed 和运行标识。
