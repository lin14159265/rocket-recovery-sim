# 接管审计与实现状态

更新日期：2026-07-14

## 产品定位

本工程是一套**公开机理代理仿真软件**。它用于研究候选闭环的因果链、状态机、通信退化、估计误差、执行机构约束和等效接触载荷，不是实际型号数字孪生，也不能输出真实型号成功率、结构裕度或工程验收结论。

## 已打通的产品闭环

```text
场景/算法/seed
    ↓
Web Worker 确定性仿真
    ↓
SimulationRun（配置、快照、遥测、事件、指标）
    ↓
播放 / 暂停 / 单帧 / 拖动回放
    ↓
Three.js 真值与估计视图 + ECharts 遥测 + 检查器
    ↓
结果 JSON 导出 / 相同 seed 重算 / 有界扰动算法对照
```

已实现：

- 启动后自动运行标称场景，不再停留在装载页面。
- 场景预设、算法切换、参数编辑和显式“应用参数并重算”。
- 播放、暂停、前后单帧、时间轴拖动和 0.25×–4× 播放速度。
- 真实状态、地面估计和预测交会位置的三维显示。
- 遥测曲线、状态机、通信统计、控制指令、最终指标与事件证据。
- 参数来源分级：公开确认、公开估计、代理标定、研究假设。
- 模型边界说明弹窗，明确可用与不可用范围。
- 相同配置与 seed 的确定性重算；场景和完整运行结果 JSON 导出。
- 运行结果携带模型版本、配置指纹和包含二者的运行标识，便于复核与追溯。
- 固定网、α–β 和预测协同的成对有界扰动对照，可主动取消批量任务。
- 单场景与批量试验均在 Web Worker 中运行，避免阻塞界面。
- Three.js 与 ECharts 按需加载，降低首屏主包体积。

## 本次修复的核心逻辑缺陷

原实现启用 `stopOnTerminal` 时，协调器可能先进入 `SECURED`，但物理稳定判据尚未写入指标；程序会提前停止并返回上一帧 `ARREST`，造成状态机、最终快照和指标不一致。

现已改为：

1. `SECURED` 只有在物理指标也确认稳定后才允许提前停止。
2. 终止时强制记录当前精确 tick，而不是复用上一张低频 UI 快照。
3. `SIMULATION_FINISHED` 事件同步写回最后一帧和 `finalSnapshot`。
4. 增加回归测试，覆盖 `stopOnTerminal` 的物理终态一致性。

- TypeScript 类型检查：通过。
- Vitest：7 个测试文件、47 个测试全部通过。
- 生产构建：通过。

## 已知限制

- 四根阻拦索采用 Kelvin–Voigt 集总等效模型，不含分段柔性绳、波传播、滑轮摩擦和局部结构应力。
- 火箭气动、推力、姿态与导航均为终端阶段代理模型。
- 姿态传感器尚未配置独立噪声参数，当前通过显式采样边界隔离真值，但仍需进一步细化。
- 平台运动是参数化低频六自由度子集，不是海况谱或船舶水动力模型。
- 有界扰动分布是研究假设；其统计结果只能用于模型内算法对照。
- 浏览器批量试验默认限制为每种算法 3–20 个样本。大样本应使用 CLI，并记录硬件、版本与耗时。
- 目前没有导入场景 JSON、实验数据库、结果签名和跨版本迁移工具。
- 当前没有对真实硬件、半实物平台或公开试验数据进行系统辨识。
