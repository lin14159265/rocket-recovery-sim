# 系统架构设计

## 目标边界

研究终端回收阶段的协同闭环，不模拟完整入轨、再入热环境和发动机内部过程。当前采用 6DoF 刚体与四根 Kelvin–Voigt 等效阻拦索，并实现预测 PD、分布式张力控制和分层约束 MPC；分段柔性绳仍保留为后续可替换实现。

## 数据与控制流

```mermaid
flowchart LR
  P[物理真值 Plant] --> S[传感器模型]
  S --> N1[火箭导航估计]
  S --> N2[平台跟踪估计]
  N1 --> R[无线代理链路]
  R --> C[交会预测与捕获协调]
  C --> R
  C --> B[现场总线代理]
  B --> W[四绳/绞盘控制]
  C --> G[火箭制导与姿态控制]
  G --> A1[推力/姿态执行器]
  W --> A2[绞盘执行器]
  A1 --> P
  A2 --> P
  P --> U[只读 UI/记录器]
```

控制器看不到真值；UI 可以同时显示真值与估计值，但没有控制写权限。

## 模块职责

- `engine`：整数 tick、确定性调度、随机数、记录和重放。
- `plant`：火箭刚体、平台运动、环境、绳索、接触和执行器。
- `sensors`：测量采样、噪声、偏差、失效和量测时间戳。
- `comms`：编码、CRC、时延队列、带宽、丢包、乱序、重复和过期拒绝。
- `estimation`：α–β 基线与常加速度 Kalman 预测模式。
- `control`：火箭终端制导、网中心/半间距预测 PD、版本绑定协同握手和接触后四绳 PI 张力控制。
- `mpc`：20 Hz 分层约束规划；使用确定性投影梯度求解，异常时回退预测 PD。
- `supervisor`：`SEARCH → TRACK → SYNC → ARMED → CLOSING → CONTACT → ARREST → SECURED`，以及 `ABORT/MISSED/BROKEN`。
- `metrics`：脱靶、相对速度、姿态、张力、表观载荷、链路和成功率。

## 模拟协议

外部可见消息至少包含：

```text
version, source, destination, type, sequence,
producedTick, expiresTick, payload, crc32
```

火箭上行：`HEARTBEAT`、`VEHICLE_STATE`、`CAPTURE_READY`。

平台下行：`CAPTURE_PLAN`、`PREPARE`、`COMMIT`、`ABORT`。

现场总线：`WINCH_COMMAND`、`WINCH_STATUS`、`TENSION_STATUS`。

迟到、重复、CRC 错误或过期命令必须被拒绝并计数。实际协议未公开，本协议仅为候选实现。

## 默认多速率

- 物理与接触：500 Hz（2 ms）。
- 传感器与火箭控制：100 Hz。
- 网/绞盘控制：100 Hz。
- 无线遥测与捕获计划：20 Hz。
- 现场总线：100 Hz。
- UI 快照：30 Hz。

所有周期都是基础 tick 的整数倍；显示掉帧不得改变物理步长。

## 状态所有权

- 仿真内核唯一拥有真值。
- 火箭节点拥有导航估计、飞控命令和本地安全状态。
- 平台跟踪器拥有目标估计与协方差。
- 捕获协调器拥有捕获窗口和双端握手。
- 网控拥有四绳位置、速度、张力与绞盘状态。
- 通信模块拥有序号、队列、超时、CRC 与统计。
- 每条控制量必须保留 `desired` 与 `applied`。

## 浏览器应用层

- `App`：持有场景配置、运行结果、回放时间、脏状态和错误状态。
- `simulation.worker`：执行单场景和算法对照；新的单场景请求会终止旧 Worker，避免过期结果覆盖当前配置。
- `Dashboard`：只消费运行结果和回调，不直接调用物理内核。
- `RecoveryScene` / `TelemetryCharts` / `AlgorithmWorkspace`：按需加载，分别负责三维、通用遥测和算法诊断，不拥有控制状态。
- 参数连续编辑只标记“待应用”；场景与算法离散切换立即重新运行，避免滑块拖动时排队大量仿真。
- 回放读取预计算快照，不重新推进物理时间，因此显示掉帧与拖动不会改变仿真结果。
